# 📖 项目简介

本项目是一套完整的**嵌入式硬件 + 上位机软件**双通道生理信号采集方案，面向生物医学工程教学、嵌入式开发与生理信号验证场景设计。

**硬件端**基于 STM32 HAL 库开发，集成 ECG 心电与 PPG 光电容积脉搏波信号采集电路；**上位机**基于 Python + PyQt5 构建图形化交互界面，通过串口与下位机通信，实现实时波形绘制、数字滤波预处理、生命体征参数解算、硬件增益调控、模拟信号生成、多格式数据导出与报警日志记录等完整功能。

# ✨ 核心功能

## 🔌 硬件采集端（STM32）

- 基于 STM32 HAL 库开发
- ECG 心电信号模拟前端采集
- PPG 光电容积脉搏波传感器驱动
- 双通道同步 ADC 采样
- 可编程增益放大器（PGA）控制
- 串口数据传输（UART）
- 低功耗设计与电源管理

## 📡 串口通信

- COM 口自动识别与手动选择
- 波特率可调（默认 115200）
- 数据位、停止位、校验位配置
- 串口状态实时显示与一键开关

## 📈 双通道波形显示

- ECG 心电信号实时波形绘制
- PPG 脉搏波信号实时波形绘制
- 双通道独立画布，网格背景
- 量程自适应，波形滚动刷新

## 🧹 数字滤波系统

### ECG 通道

- 高通滤波
- 低通滤波
- 陷波滤波
- 50Hz 工频滤波

### PPG 通道

- 高通滤波
- 低通滤波
- 陷波滤波
- 50Hz 工频滤波

两路滤波可独立开关，灵活适配不同采集环境。

## 💓 生命体征解算

- 心率（BPM）实时计算
- 血氧饱和度（SpO₂）实时解算
- 脉搏率实时监测
- 数值面板动态刷新显示

## 🎚️ 硬件增益调控

- 支持 x1 / x2 / x4 / x8 / x16 / x32 多档位增益调节
- 串口指令直接下发控制下位机硬件
- 实时响应，动态调整信号幅度

## 🔬 模拟信号模式

- 内置模拟信号发生器
- 模拟心率滑块调节（BPM）
- 模拟血氧滑块调节（%）
- 脱离硬件环境完成功能调试与方案演示

## 📤 数据导出

- 导出 CSV 格式原始数据
- 导出 Excel 格式结构化数据
- 一键生成 PDF 格式检测报告
- 支持完整采样数据与参数记录

## 🚨 报警与日志系统

- 生理参数异常事件记录
- 报警事件时间戳完整保留
- 串口通信原始数据日志
- 状态栏实时显示波特率、运行时间、数据量

# 🚀 快速开始

## 克隆项目

```bash
git clone https://github.com/ChenPinan001/ECG-PPG-Monitor.git
cd ecg-ppg-monitor
```

## 硬件端编译烧录

1. 使用 Keil MDK 打开 `Hardware/` 目录下的工程文件
2. 配置目标芯片型号与调试器
3. 编译工程并烧录至 STM32 开发板
4. 连接 ECG 电极与 PPG 传感器

## 软件端安装依赖

```bash
pip install -r requirements.txt
```

## 连接硬件

将 STM32 开发板通过 USB 转串口模块连接至电脑，确认串口号。

> 无硬件时可使用**模拟模式**进行功能体验。

## 启动上位机

```bash
python main.py
```

## 操作步骤

1. 在「串口设置」区域选择对应的 COM 口与波特率
2. 点击「打开串口」开始数据采集
3. 根据需要开启 ECG / PPG 通道的滤波选项
4. 调节增益档位适配信号幅度
5. 点击「导出 CSV / 导出 Excel / 生成 PDF」保存数据

# 📝 功能说明

## 滤波参数说明

| 滤波类型 | 作用 | 适用场景 |
| -------- | ---- | -------- |
| 高通滤波 | 去除基线漂移与低频干扰 | 运动、呼吸引起的基线偏移 |
| 低通滤波 | 抑制高频噪声与毛刺 | 肌电干扰、环境高频噪声 |
| 陷波滤波 | 去除特定频率干扰 | 工频干扰、电源噪声 |
| 50Hz 滤波 | 工频陷波专用 | 国内市电 50Hz 干扰抑制 |

## 增益档位说明

| 增益档位 | 放大倍数 | 适用信号强度 |
| -------- | -------- | ------------ |
| x1 | 1 倍 | 强信号场景 |
| x2 ~ x8 | 2 ~ 8 倍 | 常规采集场景 |
| x16 ~ x32 | 16 ~ 32 倍 | 微弱信号、低灌注场景 |

## 模拟模式使用

开启模拟模式后，可通过滑块调节模拟心率与血氧数值，系统会生成对应参数的模拟信号，用于：
- 无硬件环境下的功能调试
- 界面演示与教学展示
- 算法逻辑验证与测试
